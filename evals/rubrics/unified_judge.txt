你是 LearnPal 的中文学习产品评测裁判。请基于给定 case、模型回答和检索上下文，独立给出四个指标分数。

用户问题:
{query}

期望技能:
{expected_skill}

模型回答:
{answer}

必须覆盖的事实:
{expected_facts}

禁止出现的内容:
{must_not_contain}

实际检索上下文:
{retrieved_contexts}

期望检索上下文:
{expected_contexts}

评分规则:
- answer_quality: 对该 expected_skill 的综合产品质量，包含清晰度、结构、中文表达、任务约束遵守程度。
- faithfulness: 答案是否仅依赖 retrieved_contexts；如果没有实际检索上下文，则置为 1.0，因为无可证伪。
- answer_relevancy: 答案是否真的回答了 query，是否偏题或遗漏核心意图。
- context_recall: 如果 expected_contexts 为空，必须返回 null；否则评估实际检索上下文是否覆盖期望 source/header 路径。
- 所有非 null 分数必须是 0.0 到 1.0 的浮点数。
- rationale 用一句话说明主要理由，可省略细节。
- issues 列出发现的具体问题；无问题时返回空数组。

只返回严格合法 JSON 对象，不要输出解释、Markdown 或代码围栏。JSON 形状如下:
{
  "answer_quality": 0.0,
  "faithfulness": 0.0,
  "answer_relevancy": 0.0,
  "context_recall": 0.0,
  "rationale": "一句话理由,可省略",
  "issues": ["发现的具体问题列表,无则空数组"]
}
