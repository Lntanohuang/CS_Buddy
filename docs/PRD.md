## 1. 产品概述

### 1.1 产品背景与问题陈述

在线教育市场规模持续扩大，但现有平台普遍面临以下核心痛点：

| 痛点 | 具体表现 |
|------|---------|
| **千人一面** | 学习资源统一推送，无法匹配个体认知水平与学习风格 |
| **路径僵化** | 学习路线固定，学习者无法根据自身进度灵活调整 |
| **反馈滞后** | 学习效果评估依赖期末考试或人工批改，无法实时干预 |
| **资源碎片化** | 学习者需要在多个平台间跳转搜集资料，缺少整合式学习体验 |
| **注册门槛高** | 传统问卷式注册流程冗长枯燥，用户流失率高 |
| **辅导模式单一** | 仅支持文字交互，缺少图表、视频等多模态辅导能力 |

大语言模型（LLM）的成熟为上述问题提供了新的解决路径。智伴（ZhiBan）利用 **9 个专职智能体协作架构**，将对话式画像构建、多类型资源生成、学习路径规划、主动推送、多模态辅导和效果评估整合为一体化闭环，实现"千人千面"的自适应学习体验。

### 1.2 v2.0 五大核心需求

| 编号 | 核心需求 | 一句话描述 |
|------|---------|-----------|
| C1 | 对话式学习画像自主构建 | 不填问卷，通过 5-8 轮 AI 对话完成 ≥7 维画像初建，学习过程中持续更新 |
| C2 | 多智能体协同的资源生成 | 5 个专职 Agent 生成 5+ 种资源，全部纳入 MVP |
| C3 | 个性化学习路径规划与主动推送 | PathPlanner + 每日推荐 + 定时推送 + 召回推送 |
| C4 | 多模态智能辅导 | 文字 + Mermaid 图 + AI 生成图片 + 短视频，支持追问深化 |
| C5 | 多维度学习效果评估 | 知识掌握度 / 学习效率 / 进步趋势 / 薄弱点分析，评估即时联动画像与路径 |

### 1.3 目标用户群体

- **在校学生**（高中 / 大学）：需要针对具体科目进行系统化学习或备考的主力用户群
- **职场学习者**：在职充电、考证、技能转型的成人学习者
- **教师 / 培训师**：希望借助 AI 工具高效备课、生成教学资源的教育工作者

> ⚠️ **假设**：MVP 阶段优先支持大学生群体，科目范围限定在计算机科学和英语两个品类，后续逐步扩展。待确认具体首发科目范围。

### 1.4 产品核心价值主张

**"你的 AI 学伴，懂你的学习节奏"**

| 维度 | 价值 |
|------|------|
| 个性化 | 通过对话自然构建画像，基于 ≥7 维用户画像动态生成匹配其水平与偏好的学习资源 |
| 自适应 | 学习路径根据实时评估结果自动调整，避免"太难放弃 / 太简单无聊" |
| 一站式 | 5+ 种资源类型、路径规划、多模态辅导、效果评估闭环完成，无需多平台切换 |
| 持续进化 | 画像"随学随新"——对话记忆 + 行为数据双通道持续更新，推荐精准度持续提升 |
| 主动陪伴 | 每日推荐、定时推送、召回通知，学习不断线 |

## 2. 用户角色与使用场景

### 2.1 Persona 1：小明 — 大二计算机系学生

| 属性 | 描述 |
|------|------|
| 年龄 | 20 岁 |
| 目标 | 系统学习数据结构与算法，备战实习面试 |
| 痛点 | 教材太厚不知从哪开始；刷题没有方向，不知道薄弱环节在哪 |
| 技术素养 | 中等，能使用基本开发工具 |

**典型 User Story：**

- **US-1.1**：作为一名计算机专业学生，我希望系统能根据我当前的算法掌握程度，为我生成一份从易到难的学习路径，以便我高效备战面试。
- **US-1.2**：作为一名学生，我希望每完成一个知识点后，系统自动出一道练习题测试我的掌握情况，以便我及时查漏补缺。
- **US-1.3**：作为一名学生，我希望系统能把知识点用"类比 + 代码示例"的方式讲解，因为纯文字理论我看不进去。
- **US-1.4**（v2.0 新增）：作为一名新用户，我希望注册后通过和 AI 聊天来告诉系统我的情况，而不是填一堆无聊的问卷。
- **US-1.5**（v2.0 新增）：作为一名学生，我希望系统能用思维导图帮我梳理知识结构，用短视频讲解难点，让学习更高效。
- **US-1.6**（v2.0 新增）：作为一名学生，我三天没登录后，希望收到系统提醒，帮我重拾学习节奏。

#### F01：用户注册（简化版）

**优先级**：P0  
**核心需求**：C1 — 对话式学习画像自主构建

**描述**：用户首次进入系统时，仅需提供邮箱和密码即可完成注册。**不设任何问卷或偏好调查表**，画像构建完全由后续的引导对话（F01-B）完成。

**输入**：
- 邮箱地址
- 密码（需符合安全策略：≥8 位，含字母+数字）

**输出**：
- 用户账号创建成功
- 跳转至引导对话页面（F01-B）

**验收标准**：
1. 注册流程仅 1 步：输入邮箱 + 密码 → 注册完成，无任何额外表单
2. 注册完成后 1 秒内自动跳转至引导对话页面
3. 邮箱格式校验 + 密码强度校验，不符合时给出即时提示
4. 支持邮箱 + 密码注册；⚠️ **假设**：MVP 暂不支持第三方登录（微信 / Google），待确认
5. 已注册邮箱重复注册时给出友好提示

#### F01-B：引导对话式画像构建（v2.0 新增）

**优先级**：P0  
**对应 Agent**：ProfileAgent + OrchestratorAgent  
**核心需求**：C1 — 对话式学习画像自主构建

**描述**：新用户注册后，进入一段 **5-8 轮的引导式 AI 对话**，系统通过自然对话收集用户信息，构建初始学习画像。页面配有欢迎动画效果，营造温暖的"AI 学伴初次见面"氛围。

**欢迎对话页面设计**：
- 进入页面时播放欢迎动画（如：AI 学伴角色出现 + 打招呼动效）
- 对话气泡逐步出现，带有打字机效果
- 背景氛围轻松温暖（如：渐变背景 + 微动效）
- 进度条或步骤指示器显示对话进度

**对话内容覆盖 ≥7 个画像维度**：

| 序号 | 画像维度 | 对话示例 | 收集方式 |
|------|---------|---------|---------|
| 1 | 专业背景 | "你好！先聊聊你是做什么的吧？在读学生还是已经工作了？" | 自然语言提取 |
| 2 | 知识基础 | "关于数据结构，你目前了解多少？比如数组、链表这些概念熟悉吗？" | 自然语言提取 + 选择辅助 |
| 3 | 认知风格 | "你更喜欢先看理论再做题，还是直接上手写代码边做边学？" | 自然语言提取 |
| 4 | 易错点偏好 | "你觉得自己在哪些地方最容易犯错？比如逻辑推理、边界条件、还是概念混淆？" | 自然语言提取 |
| 5 | 学习风格 | "你更喜欢哪种学习方式？看文字讲解、看视频、做练习，还是混合？" | 快捷选项 + 自由描述 |
| 6 | 学习节奏 | "你一般每天能拿出多少时间学习？什么时间段学习效率最高？" | 自然语言提取 |
| 7 | 学习目标 | "你学这个的主要目标是什么？求职面试、考试、兴趣、还是工作需要？" | 自然语言提取 |

**对话逻辑**：
- OrchestratorAgent 负责对话流控制，ProfileAgent 负责从每轮对话中提取画像信息
- 每轮对话可覆盖 1-2 个维度，自然过渡，不生硬
- 用户可随时说"跳过"或"先不说了"，系统用默认值填充未收集的维度
- 对话结束后，系统展示一份画像摘要卡片，用户可确认或修改

**输入**：
- 用户的自然语言回复（5-8 轮）

**输出**：
- 初始学习画像（≥7 维度）
- 画像摘要卡片展示给用户确认
- 跳转至学习首页

**验收标准**：
1. 对话轮数控制在 5-8 轮，总耗时 < 3 分钟
2. 对话覆盖至少 7 个画像维度（专业背景、知识基础、认知风格、易错点偏好、学习风格、学习节奏、学习目标）
3. 欢迎页面有明显的动画/动效，含 AI 学伴角色出场动画
4. 对话支持"跳过"操作，跳过后系统使用合理默认值填充
5. 对话结束后展示画像摘要卡片，用户可一键确认或点击修改
6. 确认后 2 秒内跳转至学习首页
7. 整个引导对话过程的每轮 AI 响应时间 < 2 秒
8. 对话内容存储为长期记忆（MD 文件 + ES 向量），供后续画像更新使用

#### F02：学习画像建立与持续更新

**优先级**：P0  
**对应 Agent**：ProfileAgent  
**核心需求**：C1 — 对话式学习画像自主构建

**描述**：基于引导对话（F01-B）建立初始画像，后续通过**双通道**（Agent 对话 + 系统行为）持续更新，实现"随学随新"的画像动态维护。

**画像维度（≥7 维）**：

| 维度 | 初始来源 | 持续更新来源 | 更新频率 |
|------|---------|-------------|---------|
| 专业背景 | 引导对话 | Agent 对话中提取 | 用户主动变更 |
| 知识基础（按知识点） | 引导对话 + 首次评估 | 评估结果 + 练习正确率 | 每次评估后 |
| 认知风格 | 引导对话 | 资源交互数据分析 | 每周 |
| 易错点偏好 | 引导对话 | 评估错误分析 + 对话中提及 | 每次评估后 |
| 学习风格 | 引导对话 | 资源类型偏好统计 + 对话反馈 | 每周 |
| 学习节奏 | 引导对话 | 登录频率 + 单次学习时长 | 每日 |
| 学习目标与动机 | 引导对话 | 用户主动修改 + 对话中提取 | 用户触发 |

**双通道更新机制**：

**通道 A — Agent 对话记忆提取**：
- 在用户与任何 Agent 的对话过程中，ProfileAgent 后台监听对话流
- 使用 LLM 从对话内容中提取画像相关信息（如用户提到"我最近在准备面试"→ 更新学习目标）
- 提取的长期记忆以 Markdown 格式存储在 `profiles/{user_id}/memory.md`
- 同时生成向量嵌入存入 Elasticsearch，支持语义检索
- memory.md 文件结构示例：
  ```markdown
  # 用户长期记忆 — {user_id}
  
  ## 学习目标变更记录
  - [2026-04-13] 初始目标：备战实习面试（数据结构与算法）
  - [2026-04-20] 新增目标：学习系统设计
  
  ## 知识掌握洞察
  - [2026-04-15] 用户表示"递归总是搞混"→ 标记递归为易错点
  - [2026-04-18] 对话中展示了对链表操作的熟练理解 → 上调链表掌握度
  
  ## 学习偏好发现
  - [2026-04-14] 偏好代码示例 > 文字说明
  - [2026-04-16] 喜欢类比式讲解
  ```

**通道 B — 系统行为数据**：
- 评估结果数据：正确率、错题分布、耗时
- 资源使用数据：各类型资源的完读率、停留时长、反馈评分
- 学习模式数据：登录时间分布、单次学习时长、连续学习天数

**输入**：
- 引导对话提取的初始画像数据
- Agent 对话中的长期记忆提取
- 学习行为事件流（页面停留时长、资源点击率、练习作答数据）
- EvaluatorAgent 的评估结果

**输出**：
- 结构化用户画像 JSON（供其他 Agent 消费）
- 长期记忆 MD 文件（`profiles/{user_id}/memory.md`）
- ES 向量索引（语义检索用）
- 画像变更事件（触发路径 / 资源 / 推送重新计算）

**验收标准**：
1. 引导对话完成后 3 秒内完成初始画像生成，画像包含 ≥7 个维度
2. 通道 A：每次 Agent 对话结束后，30 秒内完成记忆提取并写入 memory.md + ES
3. 通道 B：评估完成后实时更新画像中的知识掌握度
4. 画像数据通过 API 可查，返回格式为标准 JSON Schema
5. 画像更新后，系统在下一次资源推荐 / 路径计算 / 推送生成时使用最新画像
6. 用户可在个人设置中查看画像摘要并手动修正标签
7. memory.md 文件可通过管理后台查看和导出（运维用）

#### F03：个性化学习资源生成（多智能体协同）

**优先级**：P0  
**对应 Agent**：OrchestratorAgent → DocAgent / MindmapAgent / QuizAgent / VideoAgent / CodeAgent  
**核心需求**：C2 — 多智能体协同的资源生成

**描述**：根据用户画像和当前学习进度，由 OrchestratorAgent 协调 5 个专职资源生成 Agent，实时生成匹配用户水平和偏好的多类型学习资源。所有 5 种核心资源类型均纳入 MVP。

**专职 Agent 与资源类型映射**：

| Agent | 资源类型 | 描述 | 技术方案 |
|-------|---------|------|---------|
| **DocAgent** | 课程讲解文档 | 针对单个知识点的结构化讲解，支持多种表述风格（通俗 / 学术 / 类比） | LLM 生成 Markdown |
| **MindmapAgent** | 知识点思维导图 | 知识点关联的结构化思维导图，可视化知识结构 | LLM 生成结构化 JSON/Mermaid，前端渲染 |
| **QuizAgent** | 多类型练习题 | 选择题 / 填空题 / 编程题，带详细解析和答案 | LLM 生成结构化题目 JSON |
| **VideoAgent** | 教学短视频 | AI 生成的教学视频，含动画演示和语音讲解 | Remotion（视频渲染）+ 讯飞 TTS（语音合成）+ LLM（脚本生成） |
| **CodeAgent** | 代码实操案例 | 基于真实场景的代码实操案例，含运行示例和逐步讲解 | LLM 生成可运行代码 + 注释讲解 |

**附加资源类型**：

| 资源类型 | 描述 | 实现方式 |
|---------|------|---------|
| 拓展阅读材料 | 相关论文、博客、教程链接推荐 | RAG 检索（非独立 Agent），由 OrchestratorAgent 直接调用检索服务 |

**OrchestratorAgent 协调逻辑**：
1. 接收用户学习请求或路径节点推进事件
2. 查询用户画像（学习风格偏好）
3. 决定调用哪些 Agent 及调用顺序：
   - 默认组合：DocAgent + QuizAgent（基础组合）
   - 视觉型偏好：+ MindmapAgent
   - 实践型偏好：+ CodeAgent
   - 用户主动请求或复杂概念：+ VideoAgent
4. 并行调用多个 Agent，流式返回结果
5. 可选：调用 RAG 检索补充拓展阅读

**输入**：
- 用户画像（来自 ProfileAgent）
- 当前学习路径节点（来自 PathPlannerAgent）
- 用户的即时请求（如："帮我讲讲快速排序"）

**输出**：
- 多类型结构化学习资源（Markdown / JSON / 视频 URL）
- 资源元数据（类型、难度等级、预计学习时长、关联知识点）

**验收标准**：
1. 同一知识点，不同画像的用户生成的资源内容存在明显差异（如语言风格、示例选取、资源类型组合）
2. DocAgent 文字讲解生成时间 < 8 秒（首 token 延迟 < 2 秒，采用流式输出）
3. MindmapAgent 思维导图生成时间 < 5 秒，前端正确渲染为可交互的思维导图
4. QuizAgent 练习题生成时附带标准答案和解析，支持选择题 / 填空题 / 编程题三种题型
5. VideoAgent 短视频生成时间 < 60 秒（异步生成，生成完成后推送通知），视频时长 1-5 分钟
6. CodeAgent 代码案例可直接运行，含完整注释和逐步讲解
7. OrchestratorAgent 根据用户画像自动决定资源组合，无需用户手动选择
8. 资源内容准确性由评估模块抽检，事实性错误率 < 5%
9. 用户可对生成资源进行"有用 / 没用"反馈，反馈数据回流至画像
10. 所有 5 种资源类型在 MVP 中可用

> ⚠️ **假设**：VideoAgent 使用 Remotion 做视频渲染、讯飞 TTS 做语音合成。需确认讯飞 TTS API 的调用额度和成本。  
> ⚠️ **假设**：资源生成完全由 LLM 实时生成（视频除外，视频异步），不依赖预制资源库。若后续发现高频知识点重复生成导致成本过高，考虑引入缓存机制。

#### F04：学习路径智能规划与主动推送

**优先级**：P0  
**对应 Agent**：PathPlannerAgent  
**核心需求**：C3 — 个性化学习路径规划和资源推送

**描述**：基于用户画像、目标科目的知识图谱和学习进度，生成个性化的学习路径，并根据评估反馈动态调整。新增**主动推送机制**，通过每日推荐、定时推送和召回推送，主动触达用户，保持学习连续性。

**路径规划能力（保留已有设计）**：

**输入**：
- 用户画像（来自 ProfileAgent）
- 科目知识图谱（预定义的知识点依赖关系）
- 用户选定的学习目标（如："掌握数据结构基础"）
- 评估结果（来自 EvaluatorAgent，用于路径调整）

**输出**：
- 学习路径树（有序的知识点列表，含分支和前置依赖）
- 每个节点的预估学习时长
- 路径调整建议（如："建议先回顾链表，再学习树"）

**主动推送机制（v2.0 新增）**：

| 推送场景 | 触发条件 | 推送内容 | 优先级 |
|---------|---------|---------|--------|
| 今日推荐学习 | 用户登录时 | "今日推荐学习"卡片：含推荐知识点 + 预计时长 + 资源类型预览 | P0 |
| 每日定时推送 | 到达用户设定的学习时间 | "该学习啦！今天的学习任务已准备好" + 快速开始按钮 | P0 |
| 不活跃召回推送 | 用户 ≥3 天未登录 | "你已经 X 天没学习了，上次学到了 {知识点}，继续吧！" + 进度回顾 | P0 |

**推送规则**：
- "今日推荐学习"卡片基于 PathPlannerAgent 计算的下一个学习节点 + 用户画像中的学习风格生成
- 每日定时推送时间由用户在设置中配置（默认 20:00）
- 不活跃召回推送在用户 ≥3 天未登录时触发，每日最多 1 条，连续推送不超过 3 天
- MVP 阶段仅支持**站内通知**（应用内消息中心 + 登录时弹窗）

**验收标准**：
1. 用户选定目标后，3 秒内生成完整学习路径
2. 路径中的知识点顺序尊重前置依赖关系（如：不会在用户未学"变量"前安排"函数"）
3. 当评估发现用户某知识点未达标时，自动在路径中插入补强节点
4. 用户可手动调整路径（跳过已掌握的节点、添加额外节点）
5. 路径可视化展示为时间线或树状结构
6. 用户登录时，首页显示"今日推荐学习"卡片，含推荐知识点和预计学习时长
7. 用户可设置每日学习提醒时间，到达时间后系统发送站内通知
8. 用户 ≥3 天未登录时，系统自动发送召回通知（站内消息）
9. 推送内容个性化，包含用户上次学习进度和推荐知识点
10. 推送频率可控：每日最多 1 条定时推送 + 1 条召回推送

> ⚠️ **假设**：知识图谱初期由产品团队人工维护（预定义 JSON 格式），后续迭代中考虑自动化构建。需确认首发科目的知识图谱谁来提供。  
> ⚠️ **假设**：MVP 推送渠道仅限站内通知，不含短信、邮件、微信推送。后续版本考虑扩展。

#### F05：学习效果多维评估与即时联动

**优先级**：P0  
**对应 Agent**：EvaluatorAgent  
**核心需求**：C5 — 多维度学习效果评估

**描述**：在学习过程中的关键节点自动触发评估，进行**多维度学习效果分析**，并在评估完成后**即时联动**画像更新、路径调整和推送内容重算（Level A）。

**评估触发时机**：
- 每个知识点学习完成后（mini quiz，3-5 道题）
- 学习路径阶段完成后（阶段测试，10-15 道题）
- 用户主动发起测试请求

**多维度评估体系**：

| 评估维度 | 评估方法 | 数据来源 |
|---------|---------|---------|
| **知识掌握度** | 基于评估正确率 + 遗忘曲线模型，计算每个知识点的掌握概率 | 评估作答数据 |
| **学习效率** | 单位时间内的知识点完成数 + 正确率 | 学习时长 + 评估数据 |
| **进步趋势** | 多次评估的得分变化趋势 + 相同难度题目的耗时变化 | 历史评估数据 |
| **薄弱点分析** | 错题聚类分析（按知识点、题型、错误类型分类） | 错题数据 + 行为数据 |

**即时联动机制（Level A）**：

每次评估完成后，立即触发以下三个联动动作：

```
评估完成
   │
   ├──▶ ① ProfileAgent：更新画像（知识掌握度、易错点、学习效率）
   │
   ├──▶ ② PathPlannerAgent：重新计算路径（掌握→推进 / 部分掌握→补充 / 未掌握→回退）
   │
   └──▶ ③ 推送服务：重新计算"今日推荐学习"内容
```

**输入**：
- 评估题目与用户作答数据
- 用户学习行为数据（资源停留时长、重复阅读次数）
- 用户画像（用于题目难度匹配）

**输出**：
- 多维评估报告（知识掌握度 + 学习效率 + 进步趋势 + 薄弱点分析）
- 画像更新指令（传递给 ProfileAgent）
- 路径调整指令（传递给 PathPlannerAgent）
- 推送内容重算指令

**验收标准**：
1. 评估题目难度与用户当前水平匹配（基于画像），不出现"碾压式"难题或"无意义"简单题
2. 评估结果在用户提交答案后 3 秒内返回
3. 每道错题附带错误原因分析和推荐复习知识点
4. 评估报告包含全部 4 个维度：知识掌握度、学习效率、进步趋势、薄弱点分析
5. 评估完成后 5 秒内完成画像更新 + 路径调整 + 推送重算（Level A 即时联动）
6. 阶段测试结束后自动生成评估摘要，包含进步趋势可视化（折线图/雷达图）
7. 连续 3 次高分自动提升难度，连续 2 次低分自动降低难度并插入基础巩固
8. 评估数据支持导出（供用户查看历史评估记录）

#### F06：智能对话与多模态辅导

**优先级**：P0  
**对应 Agent**：OrchestratorAgent  
**核心需求**：C4 — 多模态智能辅导

**描述**：用户通过自然语言对话与系统交互，OrchestratorAgent 理解用户意图并调度相应 Agent 完成任务。v2.0 新增**多模态辅导能力**：当学生提问时，除文字讲解外，还可提供 Mermaid 图表、AI 生成图片和短视频，并支持追问深化对话。

**支持的意图类型**：

| 意图类别 | 示例用户输入 | 调度目标 |
|---------|------------|---------|
| 学习请求 | "帮我讲讲二叉树" | DocAgent / MindmapAgent / CodeAgent |
| 路径查询 | "我接下来该学什么" | PathPlannerAgent |
| 评估请求 | "测试一下我对排序的掌握" | EvaluatorAgent + QuizAgent |
| 画像修改 | "我觉得我已经掌握链表了" | ProfileAgent |
| 进度查询 | "我这周学了多少" | 内部数据查询 |
| 视频请求 | "能不能用视频给我讲讲这个" | VideoAgent |
| 深入追问 | "能再详细讲讲第 3 步吗" / "画个图帮我理解" | 上下文感知 → 对应 Agent |
| 闲聊 / 鼓励 | "我学不下去了" | OrchestratorAgent（直接回复） |

**多模态辅导能力（v2.0 新增）**：

| 辅导模态 | 触发方式 | 技术方案 |
|---------|---------|---------|
| **文字讲解** | 默认 | LLM 生成 Markdown（已有） |
| **Mermaid 图表** | 涉及流程/结构/关系时自动生成，或用户请求 | LLM 生成 Mermaid 语法 → 前端渲染为 SVG |
| **AI 生成图片** | 复杂的架构图/示意图，文字和 Mermaid 难以表达时 | DALL-E / Stable Diffusion API → 生成图片 |
| **教学短视频** | 用户主动请求，或 OrchestratorAgent 判断知识点适合视频讲解 | VideoAgent（Remotion + 讯飞 TTS + LLM） |

**追问深化对话支持**：
- 用户可对任何回复进行追问："这个地方没看懂"、"能举个例子吗"、"画个图帮我理解"
- OrchestratorAgent 维护多轮对话上下文，理解指代关系
- 根据追问内容决定切换辅导模态（如：文字讲解不够 → 自动补充 Mermaid 图）

**输入**：
- 用户自然语言消息
- 当前会话上下文（含历史对话和已生成的资源）

**输出**：
- 自然语言回复（流式输出）
- 可能附带的多模态内容：Mermaid 图表、AI 图片、视频链接
- 结构化内容：资源卡片、路径图、评估结果

**验收标准**：
1. 意图识别准确率 > 90%（基于标注测试集）
2. 首 token 响应时间 < 1.5 秒
3. 对话支持多轮上下文理解（如用户说"再详细讲讲"，系统理解指代上一个知识点）
4. 对话界面支持 Markdown 渲染（代码块、表格、公式）+ Mermaid 图实时渲染
5. 当涉及流程 / 结构 / 关系时，自动在回复中嵌入 Mermaid 图表
6. AI 生成图片在 10 秒内返回，图片与讲解内容相关
7. 视频请求触发 VideoAgent 异步生成，用户收到"视频生成中"提示 + 完成通知
8. 追问深化对话时，系统能正确理解指代并切换辅导模态
9. 当意图不明确时，主动追问而非猜测执行

> ⚠️ **假设**：AI 生成图片（DALL-E / SD）为可选能力，MVP 阶段可降级为仅使用 Mermaid 图表。需评估图片生成 API 的成本和延迟。

## 4. 多智能体交互流程

### 4.1 Agent 总览（9 个）

| Agent | 职责 | 输入 | 输出 |
|-------|------|------|------|
| **OrchestratorAgent** | 意图识别、任务分解、Agent 调度、多轮对话管理 | 用户消息 + 会话上下文 | 组装后的用户回复（文字 + 多模态） |
| **ProfileAgent** | 画像构建（引导对话）+ 画像更新（双通道）+ 长期记忆管理 | 对话内容 + 行为数据 + 评估结果 | 用户画像 JSON + memory.md + ES 向量 |
| **PathPlannerAgent** | 学习路径生成 + 动态调整 + 推送内容计算 | 画像 + 知识图谱 + 评估结果 | 路径树 + 推荐内容 + 推送内容 |
| **EvaluatorAgent** | 题目生成 + 多维评估 + 即时联动触发 | 画像 + 作答数据 + 行为数据 | 评估报告 + 联动指令 |
| **DocAgent** | 课程讲解文档生成 | 画像 + 知识点 + 讲解风格偏好 | Markdown 讲解文档 |
| **MindmapAgent** | 知识点思维导图生成 | 画像 + 知识点 + 关联知识图谱 | Mermaid/JSON 结构化思维导图 |
| **QuizAgent** | 多类型练习题生成（选择/填空/编程） | 画像 + 知识点 + 难度要求 | 结构化题目 JSON（含答案和解析） |
| **VideoAgent** | 教学短视频生成 | 画像 + 知识点 + 脚本大纲 | 视频文件 URL + 字幕文件 |
| **CodeAgent** | 代码实操案例生成 | 画像 + 知识点 + 编程语言偏好 | 可运行代码 + 注释讲解 |

### 4.2 流程一：新用户引导对话流（Onboarding Flow）

```
新用户完成邮箱+密码注册
       │
       ▼
 ┌─────────────────┐
 │ OrchestratorAgent│  ← 检测到新用户，启动引导对话模式
 └────────┬────────┘
          │
          ├──▶ 播放欢迎动画 + AI 学伴出场
          │
          ├──▶ 开始引导对话（5-8 轮）
          │      │
          │      ├─ 第 1 轮："你好！我是你的 AI 学伴..."（专业背景）
          │      ├─ 第 2 轮："关于 XX，你目前了解多少？"（知识基础）
          │      ├─ 第 3 轮："你更喜欢怎么学？"（认知风格 + 学习风格）
          │      ├─ 第 4 轮："你觉得自己容易在哪犯错？"（易错点偏好）
          │      ├─ 第 5 轮："每天能学多久？什么时候学最高效？"（学习节奏）
          │      ├─ 第 6 轮："你学这个的目标是什么？"（学习目标）
          │      └─ 第 7-8 轮：补充 / 深化 / 自由聊天
          │
          │   每轮对话实时传递给 ProfileAgent 提取画像信息
          │
          ├──▶ ProfileAgent
          │      │
          │      ├─ 从每轮对话中提取画像维度信息
          │      ├─ 生成初始画像 JSON
          │      ├─ 创建 profiles/{user_id}/memory.md
          │      └─ 写入 ES 向量索引
          │
          ├──▶ 展示画像摘要卡片（用户确认 / 修改）
          │
          ├──▶ PathPlannerAgent
          │      │
          │      └─ 基于初始画像生成首条学习路径
          │
          ▼
    跳转至学习首页（含"今日推荐学习"卡片）
```

### 4.3 流程二：学习主流程（Learning Flow）

```
用户发送消息（如："我要开始学习数据结构"）
       │
       ▼
 ┌─────────────────┐
 │ OrchestratorAgent│  ← 意图识别 & 任务分解
 └────────┬────────┘
          │
          ├──① 查询用户画像 ──▶ ProfileAgent
          │     （含 memory.md 语义检索）  │
          │◀── 返回用户画像 JSON ──────────┘
          │
          ├──② 获取/更新学习路径 ──▶ PathPlannerAgent
          │       （携带用户画像              │
          │        + 知识图谱）               │
          │◀── 返回学习路径 + 当前节点 ──────┘
          │
          ├──③ 根据画像决定资源组合，并行调用专职 Agent
          │      │
          │      ├──▶ DocAgent → 课程讲解文档（流式输出）
          │      ├──▶ MindmapAgent → 思维导图（视觉型用户）
          │      ├──▶ QuizAgent → 练习题
          │      ├──▶ CodeAgent → 代码案例（实践型用户）
          │      └──▶ VideoAgent → 教学视频（异步，按需）
          │
          ├──④（可选）RAG 检索 → 拓展阅读材料
          │
          ▼
    组装回复，流式输出给用户
    （学习资源 + 路径概览 + 引导语）
```

**流程说明**：

1. **OrchestratorAgent** 接收用户消息，执行意图识别。判定为"学习请求"后进入学习主流程。
2. 调用 **ProfileAgent** 获取用户最新画像，包括通过 ES 语义检索 memory.md 中的相关记忆。
3. 将画像传递给 **PathPlannerAgent**，请求生成或获取已有的学习路径，确定当前节点。
4. **OrchestratorAgent** 根据用户画像中的学习风格偏好，决定调用哪些专职资源 Agent：
   - 默认：DocAgent + QuizAgent
   - 视觉型偏好：额外调用 MindmapAgent
   - 实践型偏好：额外调用 CodeAgent
   - 用户主动请求或复杂概念：触发 VideoAgent（异步）
5. 并行调用选中的 Agent，流式返回结果，组装为用户友好的回复。

```
用户提问（如："快速排序的分治过程没看懂"）

       │

       │   ├─ 涉及流程/步骤 → 生成 Mermaid 流程图
       │   ├─ 涉及结构/关系 → 生成 Mermaid 类图/树图
       │   ├─ 需要可视化 → 调用 AI 图片生成（可选）
       │   ├─ 复杂概念 → 触发 VideoAgent（异步）
       │   └─ 默认 → 文字讲解
       │
       ├──▶ DocAgent / MindmapAgent / VideoAgent（按需）
       │

       │
       ▼

       │

       │
```

### 4.6 关键决策点

| 决策点 | 触发条件 | 决策逻辑 | 结果 |
|--------|---------|---------|------|
| 是否启动引导对话 | 新用户首次登录 | 检测无画像记录 | 进入引导对话流程（F01-B） |
| 资源 Agent 组合选择 | 学习请求进入资源生成环节 | 基于画像学习风格：视觉型→+MindmapAgent，实践型→+CodeAgent | 调用对应 Agent 组合 |
| 资源难度选择 | 资源 Agent 生成资源时 | 基于画像中知识掌握度：<40% → 入门，40-70% → 进阶，>70% → 挑战 | 调整资源难度参数 |
| 辅导模态选择 | 用户提问触发辅导 | 问题涉及流程→Mermaid，涉及代码→CodeAgent，复杂概念→VideoAgent | 选择辅导模态 |
| 路径节点推进 vs 回退 | 评估完成后 | 掌握度 ≥ 70% → 推进；40-70% → 补充练习；<40% → 回退 | 动态修改路径 |
| 意图分类兜底 | OrchestratorAgent 无法识别意图 | 置信度 < 0.6 | 向用户追问澄清 |
| 资源重新生成 | 用户反馈"没用" | 同一知识点连续 2 次负面反馈 | 切换讲解风格 / 辅导模态重新生成 |
| 不活跃召回推送 | 用户 ≥3 天未登录 | 检测最后登录时间 | 发送召回通知 |

### 4.7 异常处理

| 异常场景 | 处理策略 |
|---------|---------|
| LLM 调用超时（> 15 秒） | 返回友好提示"正在努力思考中..."，自动重试 1 次。若仍超时，降级返回缓存资源（如有）或提示用户稍后再试 |
| Agent 间通信失败 | OrchestratorAgent 捕获异常，记录日志，向用户返回"系统暂时开小差了，请稍后再试" |
| VideoAgent 生成失败 | 降级为文字讲解 + Mermaid 图表，通知用户"视频生成暂时不可用" |
| 讯飞 TTS API 异常 | VideoAgent 降级为静音视频 + 字幕，或改用备选 TTS 服务 |
| AI 图片生成超时/失败 | 降级为 Mermaid 图表或文字描述 |
| 知识图谱缺失（用户请求的科目未配置） | 返回提示"该科目正在建设中"，引导用户选择已支持的科目 |
| 用户输入包含敏感/不安全内容 | OrchestratorAgent 内置内容安全过滤，拦截并提示用户调整输入 |
| 画像数据异常（字段缺失/格式错误） | ProfileAgent 使用默认值填充并记录告警日志 |
| memory.md 文件读写失败 | 降级为仅使用结构化画像 JSON，记录告警并后台修复 |
| ES 向量索引不可用 | 降级为不使用语义检索，仅使用结构化画像数据 |

### 5.2 安全与隐私

| 类别 | 要求 |
|------|------|
| 数据传输 | 全站 HTTPS，WebSocket 加密 |

### 6.1 数据采集与存储

| 数据类别 | 具体字段 | 存储位置 | 保留策略 |
|---------|---------|---------|---------|
| **用户基础信息** | 邮箱、密码哈希、注册时间、角色（学生/教师） | MySQL | 账号存续期间 |

| **画像向量索引** | memory.md 内容的向量嵌入 | Elasticsearch（向量检索） | 与 memory.md 同步更新 |
| **学习行为日志** | 页面访问、资源点击、停留时长、滚动深度、资源类型交互 | ClickHouse / MongoDB | 原始日志保留 90 天，聚合数据永久 |
| **评估数据** | 题目 ID、题型、用户答案、正确答案、耗时、评分、多维评估结果 | MySQL | 永久保留 |
| **对话记录** | 会话 ID、用户消息摘要、意图分类、Agent 调度链、辅导模态 | MySQL + 对象存储 | 完整记录保留 30 天，摘要永久 |
| **生成资源缓存** | 知识点 ID + 画像哈希 → 资源内容（文档/题目/代码/思维导图） | Redis + 对象存储 | LRU 淘汰，热门资源常驻 |
| **视频资源** | VideoAgent 生成的视频文件 + 字幕文件 + 缩略图 | 对象存储（OSS/S3） | 永久保留（用户生成），定期清理无引用资源 |
| **推送记录** | 推送类型、触发时间、推送内容、用户是否查看/点击 | MySQL | 保留 180 天 |
| **通知数据** | 站内通知消息、已读/未读状态、通知类型 | MySQL + Redis | 保留 90 天 |
| **反馈数据** | 资源 ID、反馈类型（有用/没用）、文字评价 | MySQL | 永久保留 |

> ⚠️ **假设**：对话中的用户原始消息仅保留 30 天（隐私合规），之后仅保留脱敏后的摘要和意图分类。需法务确认保留策略。  
> ⚠️ **假设**：memory.md 文件存储在服务器本地文件系统或对象存储中，需确认生产环境的存储方案（本地 vs OSS）。

### 6.2 数据驱动个性化

```
                    数据采集
                       │
        ┌──────────┬───┴───┬──────────┐
        │          │       │          │
   学习行为日志  评估数据  用户反馈  Agent对话记忆
        │          │       │          │
        └────┬─────┴───┬───┴────┬─────┘
             │         │       │
             ▼         ▼       ▼
       ProfileAgent  EvaluatorAgent
       (画像更新     (效果分析
        双通道)       多维度)
             │         │
             └────┬────┘
                  │
                  ▼
          ┌───────────────┐
          │  用户画像 JSON  │
          │  + memory.md   │
          │  + ES 向量     │
          └───────┬───────┘
                  │
       ┌──────────┼──────────┬──────────┐
       │          │          │          │
       ▼          ▼          ▼          ▼
 PathPlanner   DocAgent   QuizAgent   推送服务
  Agent        Mindmap    VideoAgent  (个性化推送)
 (路径调整)    Agent      CodeAgent
               (资源适配)
       │          │          │          │
       └──────────┴──────────┴──────────┘
                       │
                       ▼
                个性化学习体验
```

**关键数据驱动逻辑**：

1. **知识掌握度建模**：基于评估正确率 + 遗忘曲线模型（间隔重复），动态计算每个知识点的掌握概率。当概率衰减至阈值以下时，自动安排复习。
2. **学习风格适配**：统计用户对不同类型资源的完读率和反馈评分，自动调整 OrchestratorAgent 的资源 Agent 组合权重（如：优先调用 CodeAgent + DocAgent）。
3. **学习节奏预测**：基于历史登录和学习时长数据，预测用户偏好的学习时段和单次学习时长，据此调整推送时间和每日推荐内容量。
4. **难度自适应**：评估结果驱动实时难度调节——连续 3 次高分自动提升难度，连续 2 次低分自动降低难度并插入基础巩固。
5. **长期记忆语义检索**：当用户提问或进入新知识点时，ProfileAgent 通过 ES 向量检索 memory.md 中的相关记忆，为资源生成提供更精准的上下文。

### 7.3 后续迭代方向（远景）

- **AI 助教**：教师可训练专属 AI 助教，自动回答学生常见问题
- **协作学习**：支持学习小组内的 AI 辅助协作任务（如：结对编程练习）
- **自适应考试**：类 GRE 的 CAT（计算机自适应测试）模式，动态调整题目难度
- **开放平台**：允许第三方教育内容创作者接入 Agent 框架，扩展科目生态
- **多端推送**：扩展推送渠道至微信、短信、邮件
- **离线学习**：支持资源离线下载，学习进度离线记录、上线后同步

### 附录 B：术语表

| 术语 | 定义 |
|------|------|
| Agent | 基于 LLM 的自主决策单元，具备特定职责，可与其他 Agent 协作 |
| LangGraph | Python 框架，用于构建多 Agent 协作的有状态工作流 |
| 知识图谱 | 知识点之间前置依赖关系的结构化表示 |
| 用户画像 | 描述用户学习特征的多维数据集合（≥7 维度） |
| 流式输出 | 内容生成过程中逐 token 返回给前端，无需等待全部生成完毕 |
| Mini Quiz | 针对单个知识点的快速测试，通常 3-5 道题 |
| Mermaid | 基于文本的图表生成语法，支持流程图、类图、思维导图等 |
| Remotion | React 框架，用于以代码方式生成视频 |
| TTS | Text-to-Speech，文字转语音技术 |
| RAG | Retrieval-Augmented Generation，检索增强生成 |
| memory.md | 用户长期记忆文件，以 Markdown 格式存储 Agent 对话中提取的画像洞察 |
| ES | Elasticsearch，用于存储向量嵌入并支持语义检索 |
| 随学随新 | 画像随着学习过程持续更新的设计理念 |
| Level A 联动 | 评估完成后立即触发画像更新 + 路径调整 + 推送重算的即时联动机制 |

### 附录 C：v1.0 → v2.0 变更摘要

| 变更项 | v1.0 | v2.0 |
|--------|------|------|
| 注册流程 | 邮箱+密码+偏好问卷 | 邮箱+密码，问卷改为引导对话 |
| 画像构建 | 问卷 + 行为数据 | 引导对话（5-8 轮）+ 双通道更新（对话记忆 + 系统行为） |
| 画像维度 | 5 维 | ≥7 维（新增：专业背景、认知风格、易错点偏好） |
| 画像存储 | MySQL + Redis | MySQL + Redis + memory.md + ES 向量 |
| 资源生成 Agent | 1 个 ResourceGenAgent | 5 个专职 Agent（Doc/Mindmap/Quiz/Video/Code） |
| MVP 资源类型 | 知识讲解 + 练习题 + 思维导图 | 5 种全部纳入（讲解+思维导图+练习题+视频+代码案例） |
| 智能辅导 | 文字对话 | 多模态（文字+Mermaid 图+AI 图片+视频） |
| 主动推送 | 无 | 每日推荐+定时推送+召回推送 |
| 评估维度 | 得分 + 知识掌握度 + 错误分析 | 4 维（知识掌握度+学习效率+进步趋势+薄弱点分析） |
| 评估联动 | 异步更新 | Level A 即时联动（画像+路径+推送） |
| Agent 总数 | 4（Orchestrator/Profile/PathPlanner/Evaluator + ResourceGen） | 9（+Doc/Mindmap/Quiz/Video/Code） |
